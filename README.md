# Doosan-Font-Trajectory (DFT)
> **우분투(Ubuntu) 환경에서 3점 물리 캘리브레이션 및 사용자 정의 폰트(TTF/OTF) 분석을 통한 두산 협동로봇의 자동 영문 필기 시스템**
---

## 1. 시스템 설계 (System Design)

### 하드웨어 및 툴 센터 포인트(TCP) 설계
* **집게형 엔드이펙터 기반 공정:** 로봇 끝단에 고정 장착된 그리퍼(집게)를 활용하여 정해진 순서에 따라 펜을 집고, 종이 위에서 필기를 수행합니다.
* **사용자 정의 폰트 확장성:** 특정 싱글라인 폰트에 국한되지 않고, 일반 아웃라인 폰트(`.ttf`, `.otf`) 파일에서 뼈대 추출(Thinning) 알고리즘을 거쳐 원하는 글씨체를 사용할 수 있습니다.
* **필압 제어 최적화:** 높이의 물리적 오차를 극복하기 위해 힘 제어와 순응 제어를 통한 일정한 필압을 유지합니다.

## 4. 소프트웨어 의존성 (requirements.txt)

프로젝트 구동에 필요한 핵심 외부 라이브러리 목록입니다. `pip install -r requirements.txt` 명령어로 일괄 설치 가능합니다.

```
doosan-robotics>=1.2.0     # 두산 로봇 공식 파이썬 API 프레임워크
fonttools>=4.38.0          # 벡터 폰트 데이터 추출 및 파싱 유틸리티

networkx>=2.8              # 획순 트리 그래프 구조화 및 DFS 탐색 알고리즘용
matplotlib>=3.5.0          # 로봇 가동 전 우분투 화면 내부 2D 궤적 시각화 및 검증용
```

5. 실행 순서 및 스크립트 가이드 (Execution Guide)
모든 작업은 우분투 터미널에서 가상환경을 활성화한 후 순차적으로 실행(Launch)합니다.

Step 1. 가상환경 활성화 및 종속성 설치
```
source venv/bin/activate
pip install -r requirements.txt
```

Step 2. 원하는 폰트(TTF/OTF) 벡터 궤적 분석 및 데이터 빌드
사용하고자 하는 고딕체 등 일반 폰트 파일을 템플릿 폴더에 넣고 스크립트를 실행하여 로봇 모션용 3차원 점 시퀀스 배열로 규격화합니다.
```
python3 src/trajectory/build_glyphs.py
```

Step 3. 화면 기반 데이터 사전 검증 (선택 사항)
로봇을 켜기 전, 실제 필기 궤적과 자폭 정렬 및 줄바꿈 알고리즘이 잘 작동하는지 모니터 상으로 확인합니다.
```
python3 tests/test_font.py
```

Step 4. 테이블 지그 정렬 및 3점 물리 캘리브레이션
로봇을 핸드 가이딩 모드로 전환하여 종이 위의 세 모퉁이를 찍어 하드웨어 오차를 극복합니다.
```
python3 src/calibration/plane_calibrator.py
```
결과물: 법선 벡터 기반의 평면 방정식 파라미터가 config/calibration.json에 업데이트됩니다.

Step 5. 메인 자동화 시퀀서 구동 (최종 실행)
텍스트를 입력하면 로봇이 펜 픽업부터 글씨 쓰기, 도장, 종이 배출까지 전체 시퀀스를 순차 루프로 수행합니다.
```
python3 main.py
```
